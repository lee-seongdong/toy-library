# 헥사고날 아키텍처
도메인이 외부 의존성 없이 순수하게 유지되도록 설계하여, 기술 변경으로부터 비즈니스 로직을 보호하는 아키텍처  
내부(domain)와 외부를 포트로 연결하고 어댑터로 구현

## 1. 주요 구성요소
### 1.1. 포트 (Port)
애플리케이션이 외부와 통신하기 위한 인터페이스
- 인바운드 포트(Inbound Port / UseCase 인터페이스): 애플리케이션이 제공하는 기능을 정의. Application 레이어에 위치
```java
interface RegisterMemberUsecase {
    public MemberResponse register(RegisterMemberCommand command);
}
```
- 아웃바운드 포트(Outbound Port): 애플리케이션이 필요로 하는 외부 기능을 정의. Domain 레이어에 위치
```java
interface MemberRepository {
    public Member save(Member member);
}

interface NotificationPort {
    public void sendEmail(String email, String message);
}
```

### 1.2. 어뎁터 (Adapter)
포트가 정의한 인터페이스를 구현한 구현체
- 인바운드 어뎁터(Inbound Adapter): 외부에서 들어오는 요청을 처리. Controller, CLI, Message Consumer 등
- 아웃바운드 어뎁터(Outbound Adapter): 외부로 나가는 요청을 처리. DB, 외부 API, Message Producer 등

> **도메인 메서드 vs Usecase**  
> - 도메인 메서드 (비즈니스 로직)
>   - 단일 객체 내의 상태변경
>   - 비즈니스 규칙 검증
> - Usecase (애플리케이션 로직)
>   - 여러 도메인 객체를 조합
>   - 트랜잭션 경계
>   - 로직 흐름

## 2. 구조
```
Inbound Adapter
    |
    | (사용)
    ↓
Inbound Port (UseCase)
    ↑
    | (구현)
    |
Application Service (직접사용)--> Domain 
    |
    | (사용)
    ↓
Outbound Port
    ↑
    | (구현)
    |
Outbound Adapter
```
