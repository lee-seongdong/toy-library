## 1. 전통적 설계방법 + 레이어드 아키텍처
### 패키지 구조
```
com.example.blog
├── controller/          # Presentation
│   ├── PostController
│   ├── CommentController
│   ├── UserController
│   └── HashtagController
├── service/            # Application
│   ├── PostService
│   ├── CommentService
│   ├── UserService
│   └── HashtagService
├── model/              # Domain (빈약한 모델)
│   ├── Post
│   ├── Comment
│   ├── User
│   └── Hashtag
├── repository/         # Infrastructure
│   ├── PostRepository
│   ├── CommentRepository
│   ├── UserRepository
│   └── HashtagRepository
└── config/
    ├── DatabaseConfig
    ├── RedisConfig
    └── RestClientConfig
```

### 다이어그램
```
[View/JSON/REST API] 
        ↓
[Controller Layer] 
        ↓
[Service Layer] 
        ↓
[Model Layer] 
        ↓
[Repository Layer] 
        ↓
[DB/Redis/OpenSearch/External API]
```

## 3. DDD + 헥사고날 아키텍처
### 패키지 구조
```
com.example.blog
├── domain/            # Core (Domain)
│   ├── post/
│   │   ├── Post
│   │   ├── PostRepository (Port)
│   │   └── PostService
│   ├── comment/
│   │   ├── Comment
│   │   └── CommentRepository (Port)
│   └── user/
│       ├── User
│       └── UserRepository (Port)
├── application/       # Application Services
│   ├── port/
│   │   ├── in/        # Primary Ports
│   │   │   ├── CreatePostUseCase
│   │   │   └── GetPostUseCase
│   │   └── out/       # Secondary Ports
│   │       ├── PostPersistencePort
│   │       ├── CachePort
│   │       └── SearchPort
│   └── service/
│       └── PostApplicationService
└── adapter/           # Adapters
    ├── in/            # Primary Adapters
    │   ├── web/
    │   │   └── PostWebAdapter
    │   └── rest/
    │       └── PostRestAdapter
    └── out/           # Secondary Adapters
        ├── persistence/
        │   └── PostPersistenceAdapter
        ├── cache/
        │   └── RedisCacheAdapter
        ├── search/
        │   └── OpenSearchAdapter
        └── external/
            └── ExternalApiAdapter
```

### 다이어그램
```
[Web UI] [REST API] [External API]
    ↓         ↓           ↓
    [Primary Adapters (In)]
    ↓
    [Application Core (Domain + Use Cases)]
    ↓
    [Secondary Adapters (Out)]
    ↓         ↓         ↓         ↓
  [DB]   [Redis] [OpenSearch] [External]
```


## 4. DDD + 클린 아키텍처
### 패키지 구조
```
com.example.blog
├── entities/          # Enterprise Business Rules
│   ├── Post
│   ├── Comment
│   ├── User
│   └── Hashtag
├── usecases/          # Application Business Rules
│   ├── post/
│   │   ├── CreatePostUseCase
│   │   ├── GetPostUseCase
│   │   └── UpdatePostUseCase
│   ├── comment/
│   │   └── AddCommentUseCase
│   └── ports/         # Interface Adapters
│       ├── PostRepository
│       ├── CacheGateway
│       └── SearchGateway
├── adapters/          # Interface Adapters
│   ├── controllers/
│   │   ├── PostWebController
│   │   └── PostApiController
│   ├── presenters/
│   │   ├── PostWebPresenter
│   │   └── PostJsonPresenter
│   └── gateways/
│       ├── PostRepositoryImpl
│       ├── RedisCacheGateway
│       └── OpenSearchGateway
└── frameworks/        # Frameworks & Drivers
    ├── web/
    │   └── SpringWebConfig
    ├── database/
    │   └── JpaConfig
    └── external/
        └── RestClientConfig
```

### 다이어그램
```
[Frameworks & Drivers]
        ↓
[Interface Adapters]
        ↓
[Use Cases]
        ↓
[Entities]
```
