# 클린 아키텍처
계층을 4개(Entity, UseCase, Adapter, Framework)로 나누고, 의존성은 항상 안쪽을 향하도록 설계

## 1. 주요 구성요소
### 1.1. Entity (Domain)
핵심 비즈니스 규칙을 가진 가장 안쪽의 계층. 외부 의존성 없음.  
포함요소: 도메인, Entity, VO

### 1.2. UseCase
애플리케이션에 특화된 비즈니스 규칙을 가진 계층. Entity를 사용하여 애플리케이션 흐름 제어  
포함요소: UseCase(인터페이스), Interactor (UseCase 구현체), Port(인터페이스), DTO

### 1.3. Adapter
데이터 형식을 변환하는 계층. UseCase와 외부 사이의 데이터 변환을 담당  
포함요소: Controller (inbound port), Gateway (outbound port 구현체), Presenter(UseCase 응답값을 필요에 맞게 변환, REST API에서는 자주 생략)

### 1.4. Framework (Infrastructure)
가장 바깥쪽에서 세부 구현을 담당하는 계층. 프레임워크나 구체적인 기술에 의존하는 로직    
포함요소: JPA Entity, HTTP Client, DB Config 등

```
┌───────────────External─────────────────┐
|  (Browser)       (DB, External API)    |
└────────────────────────────────────────┘
    |                       ↑
    | (in)                  | (out)
    ↓                       |
┌────────────Infrastructure──────────────┐
|  (Spring MVC)       (JpaRepository)    |
|                    (ExternalApiClient) |
└────────────────────────────────────────┘
    ↑                       ↑
    | (depend)              | (depend)
    |                       |
┌───────────────Adapter──────────────────┐
|  (Controller)     (RepositoryGateway)  |
└────────────────────────────────────────┘
    |                       |
    | (depend)              | (impl)
    ↓                       ↓
┌───────────────UseCase──────────────────┐
|  (Inbound Port)       (Outbound Port)  |
| UseCase Interface     Repository, etc  |
|      ↑                      ↑          |
|      | (impl)               | (depend) |
|      |                      |          |
| ┌────────Interactor────────────┐       |
| |  Business Flow               |       |
| └──────────────────────────────┘       |
└──────┼─────────────────────────────────┘
       |
       | (use)
       ↓
┌───────────────Domain───────────────────┐
|  Pure Business Logic                   |
└────────────────────────────────────────┘
```
