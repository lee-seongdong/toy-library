# Member 컨텍스트

## 아키텍처
헥사고날 아키텍처  
의존성 방향: 외부 -> 내부

```
     ┌─────────────────┐
     │   Inbound       │
     │   Adapter       │ ──┐
     │ (Controller)    │   │
     └─────────────────┘   │
                           ▼
     ┌─────────────────────────────────┐
     │      Application                │
     │   (Use Cases + Service)         │
     └─────────────────────────────────┘
                           │
                           ▼
     ┌─────────────────────────────────┐
     │        Domain                   │ ◄─── 모든 의존성이
     │   (Entities + Outbound Ports)   │      여기를 향함
     └─────────────────────────────────┘
                           ▲
     ┌─────────────────┐   │
     │   Outbound      │   │
     │   Adapter       │ ──┘
     │ (Repository)    │
     └─────────────────┘
```

## 특징
- 포트와 어댑터: 명시적인 인터페이스 분리
- Use Case 중심: 비즈니스 기능이 명확히 드러남
- 테스트 용이성: 모든 외부 의존성이 포트로 추상화
- 확장성: 새로운 어댑터 추가가 쉬움
